About the Project

Regional Food Balance Sheet

The Regional Food Balance Sheet (RFBS) project aims to revolutionize agricultural monitoring across East Africa by integrating Earth observation (EO) data with multiple yield forecasts. Through the collaborative development of standardized, semi-automated models, the project generates foundational datasets for Maize, Rice, and Beans to address critical data gaps in regional food security. This training focuses on knowledge transfer and capacity building for national partners to co-produce accurate yield forecasts within their respective countries.

What You'll Learn

1
Crop Yield Forecasting with EO
How satellite datasuch as NDVI, rainfall, temperature,is used to predict maize yields across East Africa.
2
YieldWatch ML Pipeline
Run an end-to-end machine learning pipeline that extracts features from GEE and generates country-level yield predictions.
3
Yield Raster Map Generation
Convert yield prediction CSVs into georeferenced raster maps using Google Earth Engine,both yield and anomaly layers.
4
QGIS Map Styling
Style yield and anomaly maps with proper color ramps, administrative boundaries, labels, and legends.
5
Xylem-Autopilot Bulletin Pipeline
Use the automated pipeline to generate RFBS bulletins,from raw data to AI-written narratives and final reports.
6
Helmets Crop Data Collection
Collect and process field crop type data points using GoPro cameras,from raw photos to georeferenced crop labels.
Before You Arrive

Prerequisites

1
Google Account & Google Drive
Required

Almost everything in this training runs through Google - Colab notebooks, Earth Engine, and your exported map files all live in Google Drive. You need a Google account before anything else.

  1. Go to accounts.google.com/signup and create a free Google account if you don't already have one.
  2. Sign in and open drive.google.com - make sure you have at least 2 GB of free space for exported raster files.
💡 Use a personal Gmail if possible. Institutional Google accounts sometimes block Earth Engine access.
2
Google Earth Engine — Cloud Project, API & Registration
Required

Earth Engine requires two things working together: a Google Cloud Project (your billing/API container) and an Earth Engine registration (grants you access to the GEE compute environment). Do these in order.

Part A — Create a Cloud Project & Enable the API

  1. Go to console.cloud.google.com → sign in → click the project dropdown at the very top → New Project.
  2. Name it rfbs-yourname (lowercase, hyphens only, no spaces). Click Create and wait ~15 seconds until the notification says it's ready.
  3. Make sure the new project is selected in the top dropdown, then open the Earth Engine API page and click Enable.
  4. Copy your Project ID from the Cloud Console home page — it may look like rfbs-yourname-123456. You'll paste this into the Colab notebook's first cell where it says ee.Initialize(project='...').

Part B — Register for Earth Engine Access

  1. Go to code.earthengine.google.com/register → sign in with the same Google account → click Register a Noncommercial or Commercial Cloud project → choose Unpaid usage.
  2. On the registration form, fill in the answers exactly as shown below:
Registration Question Answer to Select / Type
Which of the following best describes you or your organization?Public or private academic institution (including faculty, staff, students)
What is the name of your academic institution?Your university or national research agency
Will you receive payment from commercial entities for applications or data created using Earth Engine?No
How would you describe your use of Earth Engine?Scientific research
What is your research question?Estimating yield and production for African countries
What is the geographic scope of your study?Global
Have you previously published work on this topic that used Earth Engine?No
Does your work with Earth Engine fall into any of these categories?Adaptation
Will you use Earth Engine for any of the following?Agriculture
  1. Click Submit. Approval is usually instant. Open code.earthengine.google.com — if a map loads, you're registered and ready.
✅ Why Noncommercial? — Free, no billing required, sufficient for all training and research workflows. The correct tier for university use.

🏢 Why Commercial later? — Once Xylem-Autopilot moves to full operational production at AGRA, a Commercial project unlocks higher compute quotas, priority API access, SLA guarantees, and the ability to build fee-for-service data products. You cannot switch back from Commercial to Noncommercial, so stay Noncommercial for now.
3
Add the agra_rag Folder to Your Google Drive
Required for Automated Pipeline

The Xylem-Autopilot notebook reads and writes to a folder called agra_rag in your Google Drive. You need your own writable copy — not just a view-only shortcut — so the notebook can save your yield maps and CSVs into it during the training session.

Adding the folder

  1. Open the agra_rag shared link provided by your trainer. It will open Google Drive showing the agra_rag folder.
  2. At the top of the page click Add shortcut to Drive → select My Drive → click Add shortcut. This makes it accessible inside Colab as /content/drive/MyDrive/agra_rag.
  3. Verify write access: Open the agra_rag folder in Drive and try creating a blank Google Doc inside it. If it lets you create, you have Editor access. If it says "View only", ask your trainer to re-share the folder with Editor permissions.

After adding — verify these key sub-folders exist

Folder path inside agra_rag Purpose
data/current_data/Your ML output CSVs are automatically copied here by Cell 4
data/maps/Generated yield and anomaly PNGs are copied here
data/historical_pdfs/Pre-loaded past AGRA bulletins — knowledge base for the AI writer
embeddings/Pre-built FAISS vector index — do not delete or move
output/bulletin.html and production charts are saved here after Cell 4

In Colab — confirm the path is reachable

  1. After mounting Drive in Cell 1, run this quick check in a new cell:
    import os; print(os.path.exists('/content/drive/MyDrive/agra_rag'))
    It should print True. If it prints False, the shortcut wasn't added to My Drive correctly — redo step 2.
  2. In Cell 4, make sure AGRA_RAG_PATH is set to "/content/drive/MyDrive/agra_rag" (the default). Do not change this unless told to.
⚠️ PermissionError in Cell 4? — This means your Drive shortcut is view-only. Two fixes: (1) Ask the trainer to re-share agra_rag with Editor access, OR (2) In Colab, run import shutil; shutil.copytree('/content/drive/MyDrive/agra_rag', '/content/agra_rag_local') in a new cell before Cell 4, then change AGRA_RAG_PATH in Cell 4 to '/content/agra_rag_local'.
4
QGIS Desktop (version 3.22 or later)
Required

QGIS is the free mapping software we use to style and finalize yield maps before they go into the bulletin. Download and install it before arriving, the installer can be large (500 MB+) so don't leave it for the day of the training.

  1. Go to qgis.org → Download.
  2. Choose the Long Term Release (LTR) version, it's the most stable. As of 2026 that is version 3.34 or later.
  3. Download the installer for your operating system (Windows, Mac, or Linux) and run it. Accept all default options.
  4. Open QGIS after installation and confirm it launches correctly. You should see the main map canvas with an empty project.
💡 If you already have QGIS installed, check Help → About QGIS to confirm the version is 3.22 or higher.
5
Laptop with Chrome & Stable Internet
Required

All notebooks run in the browser. Chrome works best with Google Colab and Earth Engine, other browsers may have display issues.

  1. Download and install Google Chrome if you don't already have it.
  2. Make sure Chrome is signed into the same Google account you used for GEE registration.
  3. Bring your laptop fully charged. A power adapter is strongly recommended, Colab sessions can run for several hours.
